# Phase 2. 분석 흐름 설계

## 파이프라인 개요

AlphaFolio의 분석 파이프라인은 6단계로 구성되며, 각 단계는 대응하는 Skills.md 문서의 규칙을 참조하여 동작합니다.

```
[데이터 입력] → [파싱/정규화] → [지표 계산] → [시각화] → [AI 인사이트] → [리포트 생성]
                  ↑                ↑              ↑            ↑              ↑
            parsing_rules.md  analysis_rules.md  viz_rules.md  insight_rules.md  report_rules.md
```

---

## Stage 1. 데이터 입력

### 입력 방식
- **CSV 업로드**: 사용자가 증권사에서 다운로드한 CSV 파일을 드래그앤드롭 또는 파일 선택으로 업로드
- **샘플 데이터**: 처음 방문한 사용자를 위한 데모용 샘플 포트폴리오 제공 (국내 주식, 미국 ETF 각 1종)
- **직접 입력**: 간단한 폼으로 종목명/수량/매수가를 수동 입력 (보조 수단)

### 지원 데이터 유형
| 유형 | 예시 | 포함 정보 |
|------|------|-----------|
| 보유 종목 | 포트폴리오 현황 CSV | 종목코드, 종목명, 수량, 매수단가, 현재가 |
| 거래 내역 | 매매 히스토리 CSV | 일자, 종목, 매수/매도, 수량, 단가 |
| 수익률 데이터 | 기간별 수익률 CSV | 일자, 종목/포트폴리오, 수익률 |

### 출력
- 원본 DataFrame (raw_df)
- 파일 메타데이터 (파일명, 크기, 행/열 수, 인코딩)

## Stage 2. 파싱 / 정규화

### 참조 Skills: `parsing_rules.md`, `data-integrity-check.md`, `currency-rules.md`

### 처리 과정
1. **인코딩 감지**: UTF-8, EUC-KR, CP949 등 자동 감지
2. **컬럼 매핑**: 증권사별 다른 컬럼명을 표준 스키마로 매핑
   - 예: "종목코드" = "종목번호" = "Symbol" = "ticker"
   - 예: "매수단가" = "평균매입가" = "Avg Cost" = "avg_price"
3. **데이터 타입 정규화**: 날짜 포맷 통일, 숫자형 변환 (쉼표 제거), 통화 단위 식별
4. **마켓 식별**: 종목코드 패턴으로 국내(6자리 숫자) / 미국(영문 ticker) 자동 분류
5. **환율 처리**: API 우선 환율 적용 (data-integrity-check.md → currency-rules.md 참조)
   - USD/KRW 환율: Open API 조회 우선, 실패 시 캐싱된 최근 값 fallback
   - 환율 출처(source: "api" / "cached" / "manual")를 메타데이터에 기록
6. **데이터 무결성 게이트**: 파싱 완료 직후 cross-validation 수행 (data-integrity-check.md)
   - 수량 × 매수가 > 0 검증
   - 중복 종목 코드 탐지
   - 날짜 역전(매수일 > 기준일) 감지
   - 통화 불일치 (KR 종목에 USD 단가) 경고

### 표준 스키마 (정규화 결과)
```
{
  "ticker": str,          # 종목코드 (005930, AAPL 등)
  "name": str,            # 종목명
  "market": str,          # KR / US
  "sector": str,          # 섹터 (자동 매핑)
  "quantity": int,        # 보유 수량
  "avg_price": float,     # 평균 매수가
  "current_price": float, # 현재가
  "currency": str,        # KRW / USD
  "date": datetime,       # 기준일
  "fx_rate": float,       # 적용 환율 (KRW/USD, KR 종목은 1.0)
  "fx_source": str        # 환율 출처 ("api" / "cached" / "manual")
}
```

### 출력
- 정규화된 DataFrame (normalized_df)
- 파싱 리포트 (매핑된 컬럼 수, 인식 실패 컬럼, 경고 사항)
- 무결성 게이트 결과 (integrity_report: 통과/경고/실패 항목 목록)
- 환율 메타데이터 (fx_meta: 출처, 적용 환율, 조회 시각)

## Stage 3. 지표 계산

### 참조 Skills: `analysis_rules.md`, `benchmark-selection.md`, `correlation-diversification.md`, `income-analysis.md`, `turnover-holding.md`, `tax-fee-impact.md`

### 종목별 지표
| 지표 | 설명 | 계산식 |
|------|------|--------|
| 수익률 | 개별 종목 수익률 | (현재가 - 매수가) / 매수가 × 100 |
| 평가금액 | 현재 보유 가치 | 현재가 × 수량 |
| 손익금액 | 절대 수익/손실 | (현재가 - 매수가) × 수량 |
| 비중 | 포트폴리오 내 비중 | 평가금액 / 총 평가금액 × 100 |

### 포트폴리오 전체 지표
| 지표 | 설명 | 계산식 |
|------|------|--------|
| 총 수익률 | 가중평균 수익률 | Σ(종목수익률 × 비중) |
| 총 평가금액 | 포트폴리오 총 가치 | Σ(평가금액) |
| 총 투자금액 | 총 매수 원금 | Σ(매수가 × 수량) |
| 총 손익 | 전체 손익 | 총 평가금액 - 총 투자금액 |

### 고급 지표 (거래 내역 존재 시)
| 지표 | 설명 |
|------|------|
| 샤프 비율 (Sharpe Ratio) | 위험 대비 초과수익 측정 |
| MDD (Maximum Drawdown) | 최대 낙폭, 리스크 지표 |
| 변동성 (Volatility) | 수익률의 표준편차 |
| 베타 (Beta) | 시장 대비 민감도 |
| 섹터별 수익률 | 섹터 분류 기준 집계 수익률 |
| 마켓별 수익률 | 국내/미국 시장별 집계 |

### 상관관계 및 분산화 분석 (correlation-diversification.md)
종목 3개 이상이고 일별 수익률 시계열이 존재할 때 계산합니다.

| 지표 | 설명 |
|------|------|
| 상관관계 매트릭스 | 종목 간 Pearson 상관계수 행렬 |
| 분산화 비율 (Diversification Ratio) | 가중 개별 변동성 합 / 포트폴리오 변동성 |
| 평균 상관계수 | 매트릭스 상삼각 평균값 (분산화 수준 요약) |

- 분산화 비율 > 1.5: 분산 효과 양호
- 평균 상관계수 > 0.8: "고상관 집중 포트폴리오" 경고 인사이트 트리거

### 소득(배당) 분석 (income-analysis.md)
배당 내역 컬럼 또는 배당금 정보가 존재할 때 계산합니다.

| 지표 | 설명 |
|------|------|
| 배당 수익률 (Dividend Yield) | 연간 배당금 / 투자 원금 × 100 |
| 수익 분해 (Return Decomposition) | 자본이득(Capital Gain) vs 배당소득 비율 |
| 배당 캘린더 | 월별 배당 수령 예정일 및 금액 |

- 배당 정보가 없을 경우: 섹션 숨김 처리 (graceful fallback)

### 매매 회전율 및 보유기간 분석 (turnover-holding.md)
거래 내역이 존재할 때 계산합니다.

| 지표 | 설명 |
|------|------|
| 포트폴리오 회전율 | 연간 매매금액 / 평균 자산 × 100 |
| 평균 보유기간 | 종목별 (매도일 - 매수일) 평균 |
| 단타 비중 | 30일 미만 보유 후 매도 거래 비율 |

- 회전율 > 200%: "잦은 매매" 경고 인사이트 트리거
- 회전율 > 500%: "과도한 단기 매매" 강경고 인사이트 트리거

### 벤치마크 자동 선택 (benchmark-selection.md)
포트폴리오 구성(마켓/섹터 비중)을 분석하여 가장 적합한 벤치마크를 자동 지정합니다.

| 조건 | 선택 벤치마크 |
|------|--------------|
| KR 비중 ≥ 70% | KOSPI |
| US 비중 ≥ 70% | S&P 500 |
| 혼합 (30~70%) | KOSPI 60% + S&P 500 40% 가중 혼합 |
| 기술주 집중 (Tech > 50%) | NASDAQ 100 추가 오버레이 |

- 자동 선택 결과를 UI에 표시 ("벤치마크: KOSPI (자동 선택)")
- 사용자가 수동으로 변경 가능 (드롭다운 제공)

### 세금/수수료 영향 시뮬레이션 (tax-fee-impact.md)
세전 수익률과 세후 실수령 수익률을 비교합니다. 검증된 한국 세율 기준 적용.

| 항목 | 적용 세율/요율 |
|------|--------------|
| 국내 주식 양도소득세 | 비과세 (소액주주 상장주식) |
| 국내 배당소득세 | 15.4% (지방세 포함) |
| 해외 주식 양도소득세 | 22% (250만원 공제 후) |
| 증권거래세 (코스피) | 0.18% (2024년 기준) |
| 증권거래세 (코스닥) | 0.18% |
| 매매 수수료 | 증권사별 설정값 (기본 0.015%) |

- 모든 KPI 수익률 지표는 세전 기준으로 표시 (세전/세후 토글 제공)
- 세후 수익률 계산은 선택적 기능 (사용자 설정 필요)

### 출력
- 종목별 지표 DataFrame (metrics_by_stock)
- 포트폴리오 요약 dict (portfolio_summary)
- 섹터/마켓 집계 DataFrame (sector_summary, market_summary)
- 상관관계 매트릭스 DataFrame (correlation_matrix) — 조건부
- 배당 분석 결과 dict (income_summary) — 조건부
- 회전율/보유기간 dict (turnover_summary) — 조건부
- 선택된 벤치마크 정보 dict (benchmark_meta)
- 세전/세후 수익률 비교 dict (tax_impact_summary) — 조건부

## Stage 4. 시각화

### 참조 Skills: `visualization_rules.md`

### 차트 자동 선택 규칙
Skills.md에 정의된 조건에 따라 데이터 특성에 맞는 차트를 자동 선택합니다.

| 데이터 조건 | 선택되는 차트 | 라이브러리 |
|-------------|---------------|------------|
| 종목별 비중 데이터 | 도넛 차트 (Pie) | Plotly |
| 섹터별 집계 | 수평 막대 차트 | Plotly |
| 기간별 수익률 추이 | 라인 차트 | Plotly |
| 종목별 수익률 비교 | 막대 차트 (정렬) | Plotly |
| 마켓별 비중 (KR/US) | 트리맵 | Plotly |
| 수익/손실 분포 | 히스토그램 또는 워터폴 | Plotly |
| 리스크 vs 수익률 | 산점도 (버블) | Plotly |
| 종목 간 상관관계 | 히트맵 (Heatmap) | Plotly |
| 월별 배당 수령 | 누적 막대 차트 (ticker별 스택) | Plotly |
| 포트폴리오 회전율 추이 | 라인 차트 (200%/500% 기준선 포함) | Plotly |

### 인터랙션 설계
- **드릴다운**: 섹터 차트 클릭 → 해당 섹터 종목 목록 표시
- **필터**: 마켓(국내/미국), 수익/손실 종목 필터링
- **기간 선택**: 1주/1개월/3개월/6개월/1년/전체 (거래 내역 존재 시)
- **호버 툴팁**: 종목명, 수익률, 평가금액 등 상세 정보

### 출력
- Streamlit 페이지에 렌더링된 인터랙티브 차트들
- 차트 구성 메타데이터 (어떤 차트가 왜 선택되었는지)

## Stage 5. AI 인사이트

### 참조 Skills: `insight_rules.md`

### 인사이트 생성 규칙 (규칙 기반 + AI 보강)

#### 규칙 기반 인사이트 (코드로 자동 감지)
| 패턴 | 인사이트 예시 |
|------|---------------|
| 특정 종목 비중 > 30% | "⚠️ {종목명} 비중이 {비중}%로 집중도가 높습니다. 분산 투자를 고려하세요." |
| 섹터 편중 (1개 섹터 > 50%) | "📊 {섹터}에 {비중}% 집중되어 있습니다. 섹터 다각화를 검토하세요." |
| MDD > 20% | "📉 최대 낙폭이 {MDD}%입니다. 리스크 관리가 필요합니다." |
| 수익률 상위/하위 종목 | "🏆 최고 수익 종목: {종목명} (+{수익률}%)" |
| 국내/해외 비중 불균형 | "🌏 해외 자산 비중이 {비중}%입니다. 글로벌 분산 수준을 점검하세요." |
| 총 수익률 > 시장 벤치마크 | "✅ 포트폴리오가 시장 대비 {차이}%p 초과 수익 중입니다." |

#### AI 보강 인사이트 (선택적 LLM 연동)
- 규칙 기반 인사이트를 종합하여 자연어 요약문 생성
- 포트폴리오 개선 제안 생성
- LLM 미연동 시에도 규칙 기반 인사이트만으로 동작 가능 (graceful fallback)

### 출력
- 인사이트 카드 목록 (insight_cards[])
- 종합 요약 텍스트 (summary_text)

## Stage 6. 리포트 생성

### 참조 Skills: `report_rules.md`

### 리포트 구성
1. **헤더**: 포트폴리오명, 기준일, 총 평가금액
2. **KPI 카드**: 총 수익률 (세전), 총 손익, 종목 수, 샤프비율
3. **차트 섹션**: Stage 4에서 생성된 주요 차트 3~5개
4. **인사이트 섹션**: Stage 5에서 생성된 핵심 인사이트
5. **상관관계 분석 섹션**: 히트맵 + 분산화 비율 해석 (종목 3개 이상, 시계열 존재 시)
6. **배당/소득 분석 섹션**: 배당 캘린더 + 수익 분해 차트 (배당 정보 존재 시)
7. **매매활동 섹션**: 회전율 추이 + 단타 비중 요약 (거래 내역 존재 시)
8. **종목 상세 테이블**: 전체 종목 목록 + 개별 지표
9. **푸터**: 면책조항 (4개 항목) + 환율 출처 표기 + 생성 시각

### 내보내기 형식
- **대시보드 내 뷰**: Streamlit 페이지로 실시간 확인
- **PDF 내보내기**: 리포트를 PDF로 다운로드 (선택 기능)

### 출력
- 완성된 대시보드 페이지
- PDF 파일 (선택)

## 데이터 흐름 다이어그램

```
사용자 CSV 업로드
    │
    ▼
[Stage 1] raw_df 생성
    │
    ▼
[Stage 2] parsing_rules.md + currency-rules.md 참조 → normalized_df 생성
    │
    ▼
[무결성 게이트] data-integrity-check.md → 통과/경고/실패 판별
    │ 실패 시 → 에러 화면 (파싱 오류 안내)
    │ 경고 시 → 계속 진행 + 경고 배너 표시
    ▼
[Stage 3] analysis_rules.md + benchmark-selection.md 참조
    │       ↳ (조건부) correlation-diversification.md
    │       ↳ (조건부) income-analysis.md
    │       ↳ (조건부) turnover-holding.md
    │       ↳ (조건부) tax-fee-impact.md
    │       → metrics_by_stock + portfolio_summary + 확장 분석 결과
    │
    ├──────────────────┐
    ▼                  ▼
[Stage 4]           [Stage 5]
viz_rules.md 참조    insight_rules.md 참조
차트 렌더링          인사이트 카드 생성
(상관관계 히트맵,     (회전율 경고,
 배당 차트,           분산화 수준 경고 포함)
 회전율 차트 포함)
    │                  │
    └──────┬───────────┘
           ▼
[Stage 6] report_rules.md 참조
    → 대시보드 (상관/배당/매매활동 섹션 포함)
    → PDF 리포트 (면책 4줄 + 환율 출처 푸터 포함)
```

※ Stage 3의 조건부 분석(상관관계/배당/회전율)은 데이터 가용성에 따라 선택적으로 실행됨
