# Phase 3. Skills.md 설계 전략

## 설계 3원칙

### 1. 선언적 (Declarative)
Skills.md는 **"무엇을"** 정의합니다. **"어떻게"**는 코드가 처리합니다.
- 예: "수익률 = (현재가 - 매수가) / 매수가 × 100" 이라고 선언하면, 코드가 이를 읽고 계산 함수를 생성

### 2. 교체 가능 (Swappable)
Skills.md 문서를 교체하면 동일한 파이프라인으로 다른 자산군에 대응합니다.
- 예: parsing_rules.md의 ticker 패턴만 바꾸면 암호화폐 데이터도 파싱 가능
- 모든 문서에 `extension_points` 섹션을 포함하여 확장 방법을 명시

### 3. 바이브코딩 친화 (Vibe-coding Friendly)
AI가 읽고 코드를 생성할 수 있는 구조화된 YAML 포맷을 사용합니다.
- 규칙은 YAML 코드블록으로 정의 → AI가 파싱하여 Python 코드로 변환 가능
- 자연어 설명 + 구조화된 규칙이 공존하는 하이브리드 포맷

## 문서 간 의존 관계

```
[Stage 1] parsing_rules.md
           → 표준 스키마(standard_schema) 정의, invested_amount 파생 필드 포함
                    ↓
[Stage 2] analysis_rules.md
           → 표준 스키마의 필드를 참조하여 기본/고급 지표 계산
                    ↓
           ┌────────┴────────┐
           ↓                 ↓
[Stage 3-A] viz_rules    [Stage 3-B] insight_rules
  → 지표명 참조하여          → 지표값 기반 패턴 감지
    차트 자동 선택              인사이트 카드 생성
           ↓                 ↓
           └────────┬────────┘
                    ↓
[Stage 4] report_rules.md
           → 차트 ID와 인사이트 카드를 참조하여 리포트 구성
```

핵심: 각 문서는 이전 단계 문서의 출력 스키마를 참조합니다. 이 의존 관계가 명확하므로 하나의 문서만 교체해도 하위 단계가 자연스럽게 대응됩니다.
