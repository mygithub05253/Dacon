# Demo Preparation

## 5분 데모 스크립트

### 0:00–0:30 — 문제 정의 (Problem Statement)

발화: "중소 반도체 제조사는 불량이 발생해야 원인을 압니다. 이미 손실이 난 후에요.
QualityLens는 공정 센서 데이터로 불량을 사전에 예측하고, 왜 그런지 설명하며, 무엇을 해야 할지 알려줍니다."

화면: 슬라이드 1장 (문제→솔루션 대비도) 또는 P1 대시보드 로딩 화면

### 0:30–1:30 — P1 Dashboard 개요 (정상 상태)

발화: "지금은 정상 운영 중입니다. 전체 상태 한눈에 보이시죠? 불량률 3.2%, 가동률 97%."

행동:
1. P1 열기 — 녹색 정상 배너 보여주기
2. 센서 게이지 3개 가리키며 "온도, 압력, 습도 모두 정상 범위"
3. 알림 패널 — "활성 알림 없음" 상태 확인

### 1:30–2:30 — P2 Predict: 이상 주입 → 실시간 예측

발화: "배치 B-247에서 온도가 갑자기 97°C로 올라갔습니다."

행동:
1. P2 열기
2. 온도 슬라이더를 97로 조정 (또는 배치 B-247 선택)
3. "예측 실행" 버튼 클릭
4. 결과: 불량 확률 82% — 빨간 게이지 등장
5. P1으로 자동 알림 전송 확인 (사이드바 알림 1건)

### 2:30–3:30 — P3 Explain: SHAP 분석

발화: "AI가 왜 불량이라고 했는지 보겠습니다."

행동:
1. P3 열기 (또는 P2에서 "AI 설명 보기" 버튼 클릭)
2. SHAP waterfall 가리키기: "온도가 가장 큰 원인, 기여도 +0.34"
3. 전역 중요도 차트: "전체 데이터에서도 온도가 1위"
4. 발화: "데이터 분석 안 해도 됩니다. AI가 원인을 바로 짚어줍니다."

### 3:30–4:30 — P4 Act: 조치 가이드 → 원클릭 수락

발화: "원인을 알았으니 뭘 해야 할지 봅니다."

행동:
1. P4 열기
2. 권고 액션 카드 3개 노출: "냉각 시스템 점검", "배치 속도 감소 15%", "QC 추가 샘플링"
3. "냉각 시스템 점검" → "수락" 버튼 클릭
4. 초록 확인 메시지 등장
5. 발화: "버튼 하나로 조치 기록이 남습니다. 엑셀 없이요."

### 4:30–5:00 — P1 회복 확인 + 클로징

발화: "조치 후 온도가 정상화됐습니다."

행동:
1. P1 열기 — 상태 배너가 "정상"으로 복귀
2. 알림 패널 클리어
3. 발화: "예측 → 설명 → 조치 → 확인. QualityLens는 이 사이클을 5분 안에 완성합니다. 감사합니다."

## Fail-Safe: 사전 녹화 백업

- 전체 데모를 1080p로 화면 녹화 (OBS 또는 Windows 캡처 도구)
- 저장 위치: `demo/demo-video-backup.mp4`
- 발표 전날 최종 녹화본 확인
- 인터넷 불가 환경이면 로컬 서버 확인 필수

## 발표 전략: QC 매니저 페르소나 스토리텔링

"김 과장님은 월요일 아침, 전날 밤 불량 배치가 출하됐다는 보고를 받습니다.
QualityLens가 있었다면, 금요일 오후에 이미 경고가 떴을 겁니다."

- 기술 설명보다 **비즈니스 임팩트** 강조
- 수치: "불량 탐지 시간 4시간 → 실시간", "원인 분석 2일 → 30초"
- 비전문가도 사용 가능 = 교육 비용 절감

## 예상 Q&A

| 질문 | 답변 핵심 |
|------|----------|
| 정확도는 얼마나 되나요? | MCC 0.XX, F1 0.XX — 불균형 데이터라 Accuracy 대신 MCC 사용 이유 설명 |
| 실제 현장 데이터 없이 어떻게 검증? | 데이콘 제공 데이터 + 도메인 이상값 시뮬레이션 |
| 다른 공정에도 적용 가능한가? | 센서 컬럼 매핑만 변경하면 확장 가능 — 아키텍처 범용성 |
| 왜 XGBoost? | 표 형태 데이터 SOTA, 빠른 추론, SHAP 호환성 |
| 실시간 처리 가능한가? | 1초 미만 추론, 스트리밍 확장 가능 구조 |
