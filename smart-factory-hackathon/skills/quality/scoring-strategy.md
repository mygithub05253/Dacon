# Scoring Strategy

## 평가 항목 매핑 (총 100점)

### 1. 문제정의 (20점) — 목표: 18+점

핵심 어필:
- **제조 현장 페인포인트 명확화**: "불량 발생 후 원인 분석 → 2일 소요" → "예방 → 실시간"
- **KPI 구체성**: 불량률 % 감소, 분석 시간 단축, 조치 반응 속도
- **대상 고객 명확화**: 데이터 분석 역량 없는 중소 제조사 QC 매니저

발표 전략:
- 슬라이드 1번에 "비용 피해 숫자" 제시 (예: 반도체 불량 1건 = 수천만원 손실)
- Before/After 대비 명확히

리스크: 문제를 너무 일반적으로 설명하면 감점. 반도체 공정 특수성 강조.

### 2. AI 활용 (25점) — 목표: 22+점 (최우선 투자)

핵심 어필:
- **XGBoost 선택 근거**: 표 형태 센서 데이터 SOTA, 빠른 추론, SHAP 호환
- **MCC 지표 사용**: 불균형 데이터(정상 >> 불량)에서 Accuracy 함정 설명, MCC의 우월성
- **SHAP 통합**: 블랙박스 → 화이트박스, 피처별 기여도 시각화
- **임계값 최적화**: 기본 0.5 아닌 Precision-Recall curve로 최적 threshold 선택

차별화 포인트:
- 단순 예측 모델이 아닌 **예측 → 설명 → 행동** 파이프라인 완성
- SHAP waterfall + global importance 이중 설명

### 3. 플랫폼 기획 (20점) — 목표: 17+점

핵심 어필:
- **5페이지 UX 설계**: 각 페이지의 사용자 의도 명확 (P1 모니터링 → P2 예측 → P3 이해 → P4 행동 → P5 기록)
- **제조 현장 UX**: 대형 터치 타깃, 3색 코딩, 2m 가독성
- **비전문가 친화**: 텍스트 최소화, 원클릭 액션, 조작 훈련 불필요

발표 시 보여줄 것: P4의 "수락" 버튼 원클릭 → 즉시 피드백 흐름

### 4. MVP 완성도 (25점) — 목표: 22+점 (최우선 투자)

핵심 어필:
- **End-to-end 동작**: raw 데이터 → 예측 → SHAP → 알림 → 조치 전 과정 라이브 시연
- **코드 품질**: utils/ 분리, 캐싱, 세션 상태 관리
- **에러 핸들링**: 빈 데이터, 누락 센서 대응

체크리스트:
- [ ] 5페이지 모두 오류 없이 렌더링
- [ ] 이상 주입 → 예측 → SHAP 자동 연결 동작
- [ ] P4 원클릭 수락 → P5 이력 기록

### 5. 발표 (10점) — 목표: 8+점

핵심 어필:
- 5분 타이밍 정확히 맞추기 (초과 = 인상 손실)
- 비전문가 청중 가정 → 기술 용어 최소화
- 데모 중 "이건 현장에서 이렇게 씁니다" 멘트로 실용성 강조

## 점수 극대화 전략

| 전략 | 적용 항목 |
|------|----------|
| MCC + SHAP 조합 강조 | AI활용(25) |
| 라이브 이상 주입 데모 | MVP완성도(25) + 발표(10) |
| QC 매니저 페르소나 스토리 | 문제정의(20) + 발표(10) |
| UX 설계 근거 설명 (44px, 2m) | 플랫폼기획(20) |

## 시간 부족 시 커트 우선순위

시간이 부족하면 아래 순서로 기능 축소:

1. **유지 필수**: P1(대시보드) + P2(예측) + P3(SHAP) — 핵심 AI 파이프라인
2. **간소화 가능**: P4(액션 카드 하드코딩으로 대체, DB 연동 없이)
3. **스킵 가능**: P5(이력 테이블 — st.dataframe 정적 mock 데이터로 대체)
4. **절대 포기 금지**: SHAP 시각화, MCC 지표, 원클릭 수락

## 경쟁자 대비 차별화 포인트

- 대부분 팀: 예측 모델 + 정확도 수치 → 끝
- QualityLens: 예측 + **왜** + **무엇을** → 완결된 의사결정 지원 루프
- 비전문가 QC 매니저도 즉시 사용 가능 → 실제 도입 가능성 높음
- 제조 현장 UX 원칙 적용 → 경쟁 팀과 시각적으로 차별화
